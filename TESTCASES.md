# TESTCASES — API микросервиса Avito QA Internship
Host: https://qa-internship.avito.com


## 1. POST /api/1/item — Создать объявление
### TC-1.1: Успешное создание (по примеру из Postman)
Предусловие: уникальный sellerId (111111–999999)
Шаги: POST /api/1/item с корректным JSON (в примере из коллекции)
Ожидание: 200, в ответе присутствует itemId (целое число)
### TC-1.2: Дублирование данных
Шаги: создать два одинаковых объявления
Ожидание: 200, itemId уникальны
### TC-1.3: Пустой title
Ожидание: 400 Bad Request
### TC-1.4: Цена = 0
Ожидание: 400 Bad Request
### TC-1.5: Некорректный sellerId (строка, слишком маленькое значение)
Ожидание: 400 Bad Request



## 2. GET /api/1/item/{itemId}
### TC-2.1: Успешное получение созданного объявления
Ожидание: 200, все поля соответствуют отправленным при создании
### TC-2.2: itemId = строка
Ожидание: 400 Bad Request
### TC-2.3: Несуществующий itemId
Ожидание: 404 Not Found



## 3. GET /api/1/items?sellerId=
### TC-3.1: Получение списка по sellerId
Предусловие: у seller есть ≥1 объявление
Ожидание: 200, массив, в котором есть наше объявление
### TC-3.2: sellerId не найден
Ожидание: 200, пустой массив
### TC-3.3: sellerId невалидного типа
Ожидание: 400



## 4. GET /api/1/stat/{itemId}
### TC-4.1: Успешная статистика
Ожидание: 200, поля views и contacts присутствуют, >= 0
### TC-4.2: Несуществующий itemId
Ожидание: 404
### TC-4.3: Невалидный itemId
Ожидание: 400
